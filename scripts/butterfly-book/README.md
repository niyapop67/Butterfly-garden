# Butterfly Garden Book — PDF ジェネレーター

Collector's Edition の1点、「Butterfly Garden Book」を生成するスクリプト。
**Webページ方式は使わず**、Node.jsで一度だけ実行して単一のPDFファイルを
作る方式。生成後はブラウザもサーバーも一切不要 — Windows/Mac/スマホ標準の
PDFビューアだけで開ける。

## 設計方針（なぜPlaywright/Puppeteerを使わないか）

Vercel上でChromiumを動かすのが困難という過去の制約に加えて、そもそも
「完全オフラインで、特殊ソフト不要」という絶対条件を満たすには、
ブラウザ経由のスクリーンショット/印刷よりも **pdf-lib で直接PDFを組み立てる**
方式の方が確実。フォント埋め込み・画像配置・PDFのしおり(bookmarks)まで
pdf-lib単体（+ @pdf-lib/fontkit）で完結する。

## 使い方

### 1. 依存関係インストール

```bash
cd scripts/butterfly-book
npm install
```

### 2. Firestoreからデータ書き出し

サービスアカウントキー（Firebase Console → プロジェクト設定 → サービス
アカウント → 新しい秘密鍵）を用意して:

```bash
GOOGLE_APPLICATION_CREDENTIALS=./serviceAccountKey.json npm run export
```

→ `entries.json` が生成される（削除依頼済みの投稿は自動的に除外）。

### 3. PDF生成

```bash
npm run build
```

→ リポジトリ直下に `butterfly-garden-book.pdf` が生成される。

動作確認用のサンプルデータで試したい場合:

```bash
npm run build:example
```

## デザイン仕様（LetterModalと合わせている点）

- 用紙: A4サイズのページ設計だが、**主な閲覧環境はスマホ画面**（印刷を前提にしていない）。
  本文・目次・送信者名などの文字サイズはスマホでの可読性を優先して設定している
- 各手紙＝1ページ以上（3000字までの長文は自動的に次ページへ続く）
- 便箋の背景グラデーション、ゴールド枠線 (#9c7238)
- 四隅装飾 (`corner_tl_new.png`)、上部の薔薇 (`rose_top.png`)、
  下部のクリスタル (`crystal_bottom.png`)
- 本文フォント: Klee One（手書き風）、挨拶文 "Dear MIKA": Cormorant
  Garamond Italic、送信者名・見出し: Noto Sans JP
- 表紙 + 目次(しおり形式のページ番号一覧) + 各手紙ページ
- **PDFのしおり(Outline)機能**を使い、目次からのジャンプおよびPDFビューア
  サイドバーからの各手紙への直接ジャンプに対応（Preview / Acrobat / 大半の
  スマホPDFアプリで動作確認済みの標準機能）

## 絵文字の扱い（重要）

MIKAさんのファンマーク（🦋 / 💕 / 🩷）が定着しているため、ニックネームやメッセージ本文の
ほとんどに絵文字が含まれる前提で設計している。

- Klee One / Noto Sans JPには絵文字グリフが存在しないため、**絵文字だけを
  Twemoji画像として文章中にインライン埋め込み**している（文字としては
  描画できないため、除去せず画像化する方式）
- 取得元は `jdecked/twemoji`（コミュニティが引き継いで更新を続けている版）。
  **`twitter/twemoji`（本家）は2023年にアーカイブされ更新が止まっており、
  2022年以降に追加された絵文字（🩷ピンクハートなど、MIKAさんの主要な
  ファンマークの1つ）が存在しない。** 本家は予備の取得元として残してあるが、
  実際にはjdecked版でほぼ解決する
- 初回生成時にネットワーク経由でTwemoji画像を取得し、`.emoji-cache/`に
  キャッシュする（2回目以降はキャッシュから読むため高速。このフォルダは
  再生成可能なキャッシュなのでgit管理対象外）
- 同じ絵文字が複数の投稿で使われても、埋め込みは1回のみ（ファイルサイズ・
  生成時間を抑える）
- ネットワークが使えない環境、またはTwemojiに存在しない稀な絵文字の場合は、
  その絵文字だけを省略する（投稿全体は問題なく生成される）
- 絵文字以外の本当に稀な非対応文字（フォントに全くグリフが存在しない記号等）
  は、安全策として無音で削除される（生成が止まることはない）

## ファイルサイズについて

実データ9件でテストした際、最初は**23MB**あったものを**888KB**まで削減した。
内訳と対策:

- **日本語フォントのサブセット化**（最大の要因）：Klee One・Noto Sans JPは
  数千字のCJKグリフを含むため、フル埋め込みだと1書体あたり8〜9MBになる。
  `embedFont(..., { subset: true })` で実際に使う文字だけを埋め込むよう変更
- **装飾画像の解像度最適化**：`public/images/decor/`のサイト用オリジナル画像
  （薔薇・クリスタル等、レスポンシブWeb表示用で1536×1024px）は、Book内では
  実寸110〜215pt程度でしか表示されないため、サイト側の画像は一切変更せず、
  PDF用に縮小した複製を`scripts/butterfly-book/assets/`に用意して
  そちらを埋め込むようにした。画像を差し替えたい場合はこのフォルダ内の
  PNGを更新すること（サイト本体の`public/images/decor/`とは別物）

## 既知の制約・今後の調整ポイント

- ボイス付き投稿は本文ページに「♪ ボイスメッセージあり」の一文のみ表示。
  実際の音声はVoice Collection（別ディバリアブル）側で扱う。
- 現状、蝶の種類(butterflyType)はページ内に表示していない（LetterModalの
  見た目自体にも表示されていないため、意図的に踏襲）。表示したい場合は
  `drawLetterEntry` に一行追加するだけで対応可能。
- ページごとの装飾スケール・余白は `generate-book.mjs` 冒頭の定数
  (`CARD_MARGIN_X`, `CARD_TOP`, `CARD_BOTTOM` 等)で調整可能。
